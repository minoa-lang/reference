# Path type
```
<path-type> := <type-path>
```

A path type isn't a type by itself, but it refers to another type, either directly or indirectly to:
- [opaque types]
- [struct types]
- [union types]
- [enum types]
- [bitfield types]
- [type aliases]
- [distinct types]
- [adapt types]
